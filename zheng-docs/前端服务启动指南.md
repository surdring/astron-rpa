## 2. 全局安装与依赖

### 2.1 安装 pnpm

```bash
npm install -g pnpm@latest
```

或使用 volta 自动管理版本（参考 `package.json` 中的 `volta` 配置）：

```bash
volta install node@22
volta install pnpm@10
```

### 2.2 安装项目依赖

进入 `frontend/` 目录执行：

```bash
cd frontend
pnpm install
```

> 若遇到 `electron` 相关安装失败，可配置镜像源：
> ```bash
>  pnpm config set electron_mirror https://npmmirror.com/mirrors/electron/
>  pnpm install
> ```

## 3. 环境变量配置

前端通过 `.env` 文件配置环境变量，位于 `frontend/packages/web-app/` 下。

### 3.1 可用配置文件

| 文件 | 用途 |
|------|------|
| `.env.saas` | SaaS 模式（默认） |
| `.env.enterprise` | 企业版模式 |
| `.env.example` | 配置模板（含 sentry 等占位项） |

### 3.2 关键变量说明

| 变量 | 说明 | 当前值 |
|------|------|--------|
| `VITE_AUTH_TYPE` | 认证服务类型 | `insforge` |
| `VITE_EDITION` | 版本类型（saas / enterprise） | 按需选择 |
| `VITE_INSFORGE_URL` | InsForge BaaS 后端地址 | `http://172.16.100.211:7130` |
| `VITE_INSFORGE_ANON_KEY` | InsForge 匿名密钥（前端 SDK 初始化用） | `anon_a88304...` |

### 3.3 切换配置文件

```bash
# 切换为 SaaS 模式
cd frontend
pnpm set-env:saas

# 切换为企业版模式
pnpm set-env:enterprise
```

> 配置文件切换本质是执行 `node ./env.js` 脚本，将 `.env.saas`（或 `.env.enterprise`）复制为 `.env`。

## 4. Web 端（开发模式）

### 启动命令

```bash
cd frontend
pnpm dev:web
```

等价于：

```bash
cd frontend
pnpm --filter @rpa/web-app run dev
```

### 访问地址

- **本地访问**：`http://localhost:1420/`
- **局域网访问**：`http://<本机IP>:1420/`（`vite.config.mts` 中 `server.host: '0.0.0.0'`）

### 启动流程

1. Vite 开发服务器启动，监听 `1420` 端口
2. 自动打开启动页面 `http://localhost:1420/boot.html`
3. 启动页检查认证状态，若未登录则展示登录表单
4. 登录成功后跳转到主应用页面 `http://localhost:1420/`

### 配置说明

- 端口固定为 `1420`，由 `vite.config.mts` 中 `server.port: 1420` 和 `server.strictPort: true` 控制
- 认证直接连接 InsForge BaaS（`172.16.100.211:7130`），无本地代理
- 多页面入口：`boot.html`（启动页）、`index.html`（主应用）、`logwin.html`（日志窗口）等

## 5. 桌面端（开发模式）

桌面端使用 Electron + `electron-vite`，同时启动 Web 渲染进程和 Electron 主进程。

### 先决条件

- 必须先完成 `pnpm install`（含 electron 二进制下载）
- 桌面端开发需要 Windows 环境

### 启动命令

```bash
cd frontend
pnpm dev:desktop
```

等价于：

```bash
cd frontend\packages\electron-app
pnpm dev
```

### 启动流程

1. `build:sdk` — 先构建 `@rpa/shared` SDK
2. `dev:web` — 并行启动 Vite 开发服务器（Web 渲染进程）
3. `dev:client` — 并行启动 `electron-vite dev --watch`（Electron 主进程）
4. Electron 窗口打开，加载 `http://localhost:1420/` 作为渲染内容

### 调试

- Electron 主进程日志：`electron-log` 输出到控制台
- 渲染进程（Vue 应用）：在 Electron 窗口中打开 DevTools（`F12`）
- 断点调试：使用 VS Code 的 `electron-vite` 调试配置

### Electron 版本

当前使用 `electron@^25.9.8`，通过 `electron-builder` 打包。

## 6. 浏览器插件

浏览器插件位于 `frontend/packages/browser-plugin/`，用于 RPA 流程录制中的浏览器元素拾取。

### 开发模式

没有独立的 dev 命令，构建后手动加载到浏览器。

### 构建

```bash
cd frontend
pnpm --filter @rpa/extension run build
```

### 构建产物

- **Chromium 构建**：`packages/browser-plugin/dist/chromium/`
- **Firefox 构建**：`packages/browser-plugin/dist/firefox/`

### 加载到浏览器

1. 打开 Chrome/Edge，进入 `chrome://extensions/`
2. 开启"开发者模式"
3. 点击"加载已解压的扩展程序"
4. 选择 `packages/browser-plugin/dist/chromium/` 目录

> 注意：浏览器插件需要与 Electron 桌面端配合使用，Web 端模式下插件功能受限。

## 9. 生产构建

### 9.1 Web 端构建

```bash
cd frontend
pnpm build:web
```

产物输出到 `frontend/packages/web-app/dist/`。

### 9.2 桌面端构建

```bash
# 完整构建（包含引擎 + 前端）
cd 项目根目录
./build.bat --python-exe "C:\Program Files\Python313\python.exe"

# 仅构建前端桌面端
./build.bat --skip-engine

# 或从 frontend 目录手动构建
cd frontend
pnpm build:desktop
```

桌面端构建产物输出到 `frontend/packages/electron-app/dist/`（安装包格式）。

### 9.3 构建参数说明（`build.bat`）

| 参数 | 说明 |
|------|------|
| `--python-exe <path>` | 指定 Python 解释器路径 |
| `--sevenz-exe <path>` | 指定 7-Zip 路径 |
| `--skip-engine` | 跳过引擎构建 |
| `--skip-frontend` | 跳过前端构建 |

## 10. 常见问题

### 10.1 `pnpm install` 安装 electron 失败

**现象**：`electron` 下载超时或失败。

**解决**：

```bash
# 设置镜像源
pnpm config set electron_mirror https://npmmirror.com/mirrors/electron/
# 删除 node_modules 重新安装
rm -rf node_modules packages/*/node_modules
pnpm install
```

### 10.2 前端启动后卡在启动页（Boot）

**现象**：浏览器打开后一直显示启动页，无法进入主应用。

**排查步骤**：

1. 检查浏览器 DevTools 控制台是否有网络错误
2. 确认 `VITE_INSFORGE_URL` 配置的服务器 `172.16.100.211:7130` 可访问
3. 确认 `VITE_AUTH_TYPE=insforge` 已正确设置
4. 若在浏览器开发模式下，检查登录后是否正常跳转

### 10.3 类型检查报错

**现象**：`pnpm dev:web` 或 `pnpm build` 时出现 TypeScript 类型错误。

**解决**：

```bash
# 单独运行类型检查
cd frontend
pnpm --filter @rpa/shared exec vue-tsc --noEmit --skipLibCheck
pnpm --filter @rpa/web-app exec vue-tsc --noEmit --skipLibCheck
```

### 10.4 `@rpa/shared` 包变更未生效

**现象**：修改了 `@rpa/shared` 后，Web 端未反映变化。

**解决**：重启 dev server：

```bash
# 停止 dev server 后重新启动
pnpm dev:web
```

### 10.5 桌面端开发时 Electron 窗口空白

**现象**：Electron 窗口打开后为白屏。

**排查步骤**：

1. 确认 Vite 开发服务器（`:1420`）已启动
2. 检查 Electron 主进程控制台是否有错误日志
3. 确认 `electron-vite` 配置中的渲染进程 URL 正确

### 10.6 跨域问题（CORS）

**现象**：浏览器控制台报 CORS 错误。

**说明**：前端直接请求 `172.16.100.211:7130`，如果该服务器未配置 CORS 头，浏览器会拦截请求。

**解决**：确认 InsForge BaaS 服务端已配置允许前端来源的 CORS 策略。
