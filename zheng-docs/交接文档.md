## 三、关键技术决策

1. **无 Docker 原则**：所有后端服务不得依赖 Docker，使用 systemd 管理
2. **前端直连**：前端通过 `@insforge/sdk` 直连 `:7130`，不走网关
3. **引擎直连**：Python 引擎通过 `insforge_client.py`（httpx）直连 InsForge API
4. **认证**：统一使用 InsForge Auth SDK（`signInWithPassword` 等），废弃 Casdoor
5. **实时通信**：WebSocket 改为 InsForge Realtime（Socket.IO）
6. **文件存储**：使用 InsForge Storage，buckets：`rpa-shared`、`rpa-files`（私有）、`rpa-public`（公开）
7. **服务端只做存储和轻量编排**，计算密集型任务（RPA 执行、AI 推理）全部在客户端完成

## 七、注意事项

1. 前端启动后若卡在 boot 页面，检查认证流程是否已迁移到 InsForge Auth SDK
2. 浏览器模式下 F12 被拦截（`event.ts` 中 `fkeys` 阻止了 F12），需注释掉 `'f12'`
3. `listenEvent is not supported in browser environment` 警告可忽略
4. AI 服务（ai-service）部署在 `172.16.100.211` 上，通过 `AUTH_VERIFY_MODE=remote` 验证 JWT
5. 遗留的旧 API 调用（`/api/robot/...`）会返回 404，需全部迁移到 InsForge SDK
