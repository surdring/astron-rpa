## 1. 项目概述

### 1.1 背景

AstronRPA 原有的后端架构包含 5 个独立 Java 微服务（robot-service、resource-service、openapi-service、ai-service、gateway）和 MySQL 数据库，通过 Casdoor 认证和 OpenResty 网关通信。这种架构存在部署复杂、资源占用高、维护成本大等问题。

### 1.2 目标

将 AstronRPA 后端架构从 **"Casdoor 认证 + 5 个独立微服务 + MySQL + OpenResty 网关"** 替换为 **InsForge BaaS 统一平台 + PostgreSQL**，实现后端轻量化。

### 1.3 改造原则

- **后端下沉**：业务逻辑从 Java 微服务迁移到 InsForge Edge Functions 和 PostgreSQL
- **前端直连**：前端和引擎通过 InsForge SDK 直接操作数据库，不再经过 Java 服务
- **逐步替换**：先完成数据库层和 API 层，再逐步迁移前端调用
- **可回退**：Java 服务保留，前端可通过配置切换调用目标

## 2. 架构说明

### 2.1 改造前架构

```
客户端/引擎
    ↓
OpenResty 网关（:13159）
    ↓
robot-service (Java)  ──→ MySQL
resource-service (Java) ──→ MySQL
openapi-service (Python)──→ MySQL
ai-service (Python)     ──→ 自有存储
gateway (Java/WS)       ──→ 各服务
    ↓
Casdoor 认证服务
```

### 2.2 改造后架构

```
客户端/引擎
    ↓
InsForge BaaS 平台（:7130）
    ├── PostgreSQL（:5432）←── PostgREST（:5430）
    ├── Edge Functions 运行时（Deno :7133）
    │   ├── rpa-terminal（终端注册/心跳）
    │   ├── rpa-execution（执行状态上报）
    │   ├── rpa-crud（机器人CRUD/流程/组件/变量）
    │   ├── notify（通知发送）
    │   └── blacklist（黑名单检查）
    ├── Storage（对象存储）
    │   ├── rpa-files（私有 bucket）
    │   └── rpa-public（公开 bucket）
    └── Auth（JWT 认证）
        ↓
ai-service（Python FastAPI :8001）←── 直连（AI 推理/OCR/CUA）
```

### 11.3 创建 Storage bucket

```bash
# 获取管理员 token
ADMIN_TOKEN=$(curl -s http://localhost:7130/api/auth/admin/sessions \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}' \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['accessToken'])")

# 创建私有 bucket
curl -X POST "http://localhost:7130/api/storage/buckets" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -d '{"bucketName":"rpa-files","isPublic":false}'

# 创建公开 bucket
curl -X POST "http://localhost:7130/api/storage/buckets" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -d '{"bucketName":"rpa-public","isPublic":true}'
```
