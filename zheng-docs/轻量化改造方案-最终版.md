## 1. 对 Gemini 版方案的评估

### 1.1 优点（保留）

| 优点 | 说明 |
|------|------|
| 架构映射矩阵清晰 | 将 AstronRPA 核心组件与 InsForge 能力一一对应 |
| 代码示例具体可操作 | TypeScript SDK 调用示例直接可用 |
| AI 组件下沉策略正确 | Python 强依赖组件（OCR、CUA、视觉）继续在客户端引擎执行 |
| 改造前后对比直观 | 清晰展示了从"重型微服务"到"极简 BaaS"的收益 |

### 1.2 不足（本次修正）

| 不足 | 本方案改进 |
|------|-----------|
| 未覆盖全部 5 个微服务 | 逐服务分析迁移策略，给出完整映射 |
| openapi-service 的 WebSocket 迁移过于简化 | 详细分析 workflow 执行流式推送的替代方案 |
| 缺少 MCP 协议迁移方案 | openapi-service 的 `/mcp` 端点与 InsForge 原生 MCP 的整合 |
| 缺少 scheduler/trigger 引擎服务迁移 | 分析引擎 5 个服务的 Server 端依赖，明确哪些保留在客户端 |
| 无前端改动分析 | 覆盖 web-app、electron-app、browser-plugin 的改动范围 |
| 无数据迁移策略 | MySQL → PostgreSQL 的 pgloader 方案 |
| 无分阶段计划和风险分析 | 分 4 阶段，每阶段有验收标准和回滚方案 |

## 2. 完整架构映射矩阵

### 2.1 基础设施层

| AstronRPA 组件 | 部署方式 | InsForge 替代 | 说明 |
|:---|:---|:---|:---|
| **MySQL 8.4.6** | Docker 容器 | **PostgreSQL 18** | 已部署在 `172.16.100.211:5432`，含 pgvector/pg_cron/pgcrypto/http 扩展 |
| **Redis (Bitnami)** | Docker 容器 | **不需要** | InsForge 基于 JWT + PG 内存，无需独立缓存 |
| **MinIO** | Docker 容器 | **InsForge Storage** | 内置 S3 兼容存储，通过 Dashboard 创建 `rpa-resources` 桶 |
| **OpenResty (Nginx+Lua)** | Docker 容器 | **InsForge Backend `:7130`** | 单一端口对外，无需前置网关和 Lua 认证脚本 |
| **Casdoor v2.67** | Docker 容器 | **InsForge Auth** | 内置 JWT + OAuth + RLS，无需独立认证中心 |
| **Atlas** | Docker 容器 | **InsForge 迁移系统** | 内置数据库迁移管理，`npm run migrate:up:local` |

### 2.2 微服务层

| AstronRPA 微服务 | 端口 | 技术栈 | InsForge 替代 | 迁移策略 |
|:---|:---|:---|:---|:---|
| **robot-service** | 8040 | Spring Boot 2.3.11 + MyBatis-Plus | **PostgREST + InsForge DB API** | 所有 CRUD 由 PostgREST 自动生成；定时任务由 InsForge Schedules 接管 |
| **resource-service** | 8030 | Spring Boot 3.2.4 + S3 SDK | **InsForge Storage** | 文件上传/下载/共享全部由 InsForge Storage API 替代 |
| **rpa-auth** | 10251 | Spring Boot 2.3.11 + Spring Cloud | **InsForge Auth** | JWT 认证 + RLS 行级安全 + OAuth 支持 |
| **ai-service** | 8010 | Python FastAPI | **InsForge AI 网关 + 客户端下沉** | 大模型对话走 InsForge AI（OpenRouter）；OCR/CUA 视觉下沉到客户端引擎 |
| **openapi-service** | 8020 | Python FastAPI | **InsForge Realtime + MCP + SDK** | Workflow CRUD → PostgREST；WebSocket → Socket.IO；MCP → InsForge 原生 MCP |

### 2.3 引擎服务（客户端保留）

| 引擎服务 | 是否依赖 Server | 迁移策略 |
|:---|:---|:---|
| **scheduler** | 依赖 robot-service（拉取任务、上报状态） | 改为通过 InsForge SDK 查询/更新 `tasks` 表 |
| **executor** | 依赖 resource-service（下载脚本）、robot-service（上报日志） | 改为通过 InsForge Storage 下载脚本、SDK 写入执行日志 |
| **picker** | 无 Server 依赖 | 无需改动 |
| **trigger** | 依赖 robot-service（注册触发器） | 改为通过 InsForge SDK 管理触发规则 |
| **browser-bridge** | 依赖 openapi-service（WebSocket） | 改为通过 InsForge Realtime（Socket.IO）通信 |

- 从 Nginx 网关路由 → **客户端直连 InsForge API**

**迁移脚本示例（`001_create_rpa_tables.sql`）：**

```sql
-- ============================================
-- AstronRPA 核心业务表迁移
-- 放置在 backend/src/infra/database/migrations/
-- ============================================

-- 1. 机器人（Agent）表
CREATE TABLE public.robots (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(255) NOT NULL,
    agent_key VARCHAR(255) UNIQUE,
    status VARCHAR(50) DEFAULT 'offline',
    host_name VARCHAR(255),
    ip_address VARCHAR(45),
    os_version VARCHAR(100),
    version VARCHAR(50),
    last_heartbeat TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW(),
    user_id UUID REFERENCES auth.users(id)
);

-- 2. 项目表
CREATE TABLE public.projects (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(255) NOT NULL,
    description TEXT,
    status VARCHAR(50) DEFAULT 'active',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW(),
    user_id UUID REFERENCES auth.users(id)
);

-- 3. 流程（Workflow）表
CREATE TABLE public.workflows (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(255) NOT NULL,
    description TEXT,
    project_id UUID REFERENCES public.projects(id),
    workflow_data JSONB,
    version INTEGER DEFAULT 1,
    status VARCHAR(50) DEFAULT 'draft',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW(),
    user_id UUID REFERENCES auth.users(id)
);

-- 4. 任务表
CREATE TABLE public.tasks (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(255) NOT NULL,
    workflow_id UUID REFERENCES public.workflows(id),
    robot_id UUID REFERENCES public.robots(id),
    trigger_type VARCHAR(50) DEFAULT 'manual',
    cron_expression VARCHAR(100),
    status VARCHAR(50) DEFAULT 'pending',
    priority INTEGER DEFAULT 0,
    scheduled_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW(),

-- 5. 执行记录表
CREATE TABLE public.executions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    task_id UUID REFERENCES public.tasks(id),
    robot_id UUID REFERENCES public.robots(id),
    workflow_id UUID REFERENCES public.workflows(id),
    status VARCHAR(50) DEFAULT 'running',
    start_time TIMESTAMPTZ DEFAULT NOW(),
    end_time TIMESTAMPTZ,
    duration_ms INTEGER,
    result JSONB,
    error_message TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    user_id UUID REFERENCES auth.users(id)
);

-- 6. 执行日志表
CREATE TABLE public.execution_logs (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    execution_id UUID REFERENCES public.executions(id),
    level VARCHAR(20) DEFAULT 'info',
    message TEXT,
    component_name VARCHAR(100),
    logged_at TIMESTAMPTZ DEFAULT NOW()
);

-- 7. 组件市场表
CREATE TABLE public.market_components (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(255) NOT NULL,
    description TEXT,
    category VARCHAR(100),
    version VARCHAR(50),
    package_name VARCHAR(255),
    is_public BOOLEAN DEFAULT false,
    download_count INTEGER DEFAULT 0,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    user_id UUID REFERENCES auth.users(id)
);

-- 8. 触发器表
CREATE TABLE public.triggers (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(255) NOT NULL,
    type VARCHAR(50) NOT NULL,
    config JSONB,
    task_id UUID REFERENCES public.tasks(id),
    enabled BOOLEAN DEFAULT true,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW(),
    user_id UUID REFERENCES auth.users(id)
);

-- ============================================
-- RLS 行级安全策略
-- ============================================
ALTER TABLE public.robots ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.projects ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.workflows ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.tasks ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.executions ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.execution_logs ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.market_components ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.triggers ENABLE ROW LEVEL SECURITY;

-- 通用 RLS：用户只能看自己的数据
CREATE POLICY "Users can only access their own data" ON public.robots
    FOR ALL USING (auth.uid() = user_id);
CREATE POLICY "Users can only access their own data" ON public.projects
    FOR ALL USING (auth.uid() = user_id);
CREATE POLICY "Users can only access their own data" ON public.workflows
    FOR ALL USING (auth.uid() = user_id);
CREATE POLICY "Users can only access their own data" ON public.tasks
    FOR ALL USING (auth.uid() = user_id);
CREATE POLICY "Users can only access their own data" ON public.executions
    FOR ALL USING (auth.uid() = user_id);
CREATE POLICY "Users can only access their own data" ON public.execution_logs
    FOR ALL USING (auth.uid() = user_id);
CREATE POLICY "Users can only access their own data" ON public.market_components
    FOR ALL USING (auth.uid() = user_id);
CREATE POLICY "Users can only access their own data" ON public.triggers
    FOR ALL USING (auth.uid() = user_id);

-- 组件市场公开数据可被所有用户查看
CREATE POLICY "Public components are viewable by all" ON public.market_components
    FOR SELECT USING (is_public = true);

-- ============================================
-- 索引
-- ============================================
CREATE INDEX idx_robots_user_id ON public.robots(user_id);
CREATE INDEX idx_robots_status ON public.robots(status);
CREATE INDEX idx_projects_user_id ON public.projects(user_id);
CREATE INDEX idx_workflows_project_id ON public.workflows(project_id);
CREATE INDEX idx_workflows_user_id ON public.workflows(user_id);
CREATE INDEX idx_tasks_workflow_id ON public.tasks(workflow_id);
CREATE INDEX idx_tasks_robot_id ON public.tasks(robot_id);
CREATE INDEX idx_tasks_status ON public.tasks(status);
CREATE INDEX idx_tasks_user_id ON public.tasks(user_id);
CREATE INDEX idx_executions_task_id ON public.executions(task_id);
CREATE INDEX idx_executions_robot_id ON public.executions(robot_id);
CREATE INDEX idx_executions_user_id ON public.executions(user_id);
CREATE INDEX idx_execution_logs_execution_id ON public.execution_logs(execution_id);
CREATE INDEX idx_market_components_user_id ON public.market_components(user_id);
CREATE INDEX idx_triggers_task_id ON public.triggers(task_id);
CREATE INDEX idx_triggers_user_id ON public.triggers(user_id);

-- ============================================
-- 实时通知触发器（PostgreSQL LISTEN/NOTIFY）
-- ============================================
CREATE OR REPLACE FUNCTION public.notify_execution_change()
RETURNS TRIGGER AS $$
BEGIN
    PERFORM pg_notify('realtime_message', json_build_object(
        'table', TG_TABLE_NAME,
        'schema', TG_TABLE_SCHEMA,
        'event', TG_OP,
        'id', NEW.id
    )::text);
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER execution_change_trigger
    AFTER INSERT OR UPDATE ON public.executions
    FOR EACH ROW EXECUTE FUNCTION public.notify_execution_change();
```

### 阶段二：API 层替换（预计 3-5 天）

**目标：** 前端所有 Axios 调用替换为 InsForge SDK，按模块逐个迁移。

#### 2.1 封装 InsForge SDK 客户端

在前端项目中创建 `src/insforge/` 目录：

```typescript
// frontend/packages/web-app/src/insforge/index.ts
import { createClient } from '@insforge/sdk';

const INSFORGE_BASE_URL = import.meta.env.VITE_INSFORGE_BASE_URL || 'http://172.16.100.211:7130';
const INSFORGE_ANON_KEY = import.meta.env.VITE_INSFORGE_ANON_KEY || '';

export const insforge = createClient({
  baseUrl: INSFORGE_BASE_URL,
  anonKey: INSFORGE_ANON_KEY,
});

// 认证相关
export const insforgeAuth = {
  signUp: (email: string, password: string) =>
    insforge.auth.signUp({ email, password }),
  signIn: (email: string, password: string) =>
    insforge.auth.signInWithPassword({ email, password }),
  signOut: () => insforge.auth.signOut(),
  getUser: () => insforge.auth.getUser(),
};

// 数据库操作（带 RLS 自动隔离）
export const db = {
  robots: () => insforge.database.from('robots'),
  projects: () => insforge.database.from('projects'),
  workflows: () => insforge.database.from('workflows'),
  tasks: () => insforge.database.from('tasks'),
  executions: () => insforge.database.from('executions'),
  executionLogs: () => insforge.database.from('execution_logs'),
  marketComponents: () => insforge.database.from('market_components'),
  triggers: () => insforge.database.from('triggers'),
};

// 存储操作
export const storage = {
  upload: (path: string, file: File) =>
    insforge.storage.from('rpa-resources').upload(path, file),
  download: (path: string) =>
    insforge.storage.from('rpa-resources').download(path),
  getPublicUrl: (path: string) =>
    insforge.storage.from('rpa-resources').getPublicUrl(path),
};

// 实时订阅
export const realtime = {
  executionChannel: () =>
    insforge.realtime.channel('rpa-executions'),
  taskChannel: () =>
    insforge.realtime.channel('rpa-tasks'),
};
```

#### 2.3 关键迁移示例：实时执行日志

**改造前（openapi-service WebSocket）：**

const ws = new WebSocket('ws://172.16.100.211:32742/api/rpa-openapi/ws');
ws.onmessage = (event) => {

**改造后（InsForge Realtime）：**
```typescript
import { realtime } from '@/insforge';

const channel = realtime.executionChannel();

channel
  .on('postgres_changes', {
    event: 'INSERT',
    schema: 'public',
    table: 'execution_logs',
    filter: `execution_id=eq.${currentExecutionId}`
  }, (payload) => {
    console.log('新日志:', payload.new.message);
  })
  .on('postgres_changes', {
    event: 'UPDATE',
    schema: 'public',
    table: 'executions',
    filter: `id=eq.${currentExecutionId}`
  }, (payload) => {

  })

```

### 阶段三：客户端引擎适配（预计 2-3 天）

**目标：** 引擎中的 scheduler、executor、trigger、browser-bridge 改为通过 InsForge SDK 与后端通信。

#### 3.1 引擎侧 SDK 封装

引擎是 Python 项目，有两种方式调用 InsForge API：

**方案 A：Python 直接调用 REST API（推荐）**
```python
# engine/shared/insforge_client.py
import httpx
import os

class InsForgeClient:
    def __init__(self):
        self.base_url = os.getenv("INSFORGE_BASE_URL", "http://172.16.100.211:7130")
        self.anon_key = os.getenv("INSFORGE_ANON_KEY", "")
        self.access_token = None

    def sign_in(self, email: str, password: str):
        resp = httpx.post(f"{self.base_url}/api/auth/signin", json={
            "email": email, "password": password
        }, headers={"x-api-key": self.anon_key})
        data = resp.json()
        self.access_token = data["access_token"]
        return data

    def _headers(self):
        return {
            "Authorization": f"Bearer {self.access_token}",
            "x-api-key": self.anon_key,
            "Content-Type": "application/json"
        }

    # === 机器人管理 ===
    def register_robot(self, robot_data: dict):
        return httpx.post(f"{self.base_url}/api/rest/robots",
                         json=robot_data, headers=self._headers())

    def update_robot_status(self, robot_id: str, status: str):
        return httpx.patch(f"{self.base_url}/api/rest/robots",
                          json={"status": status},
                          headers={**self._headers(), "Prefer": "return=representation"},
                          params={"id": f"eq.{robot_id}"})

    def send_heartbeat(self, robot_id: str):
        return httpx.patch(f"{self.base_url}/api/rest/robots",
                          json={"last_heartbeat": "now()"},
                          headers={**self._headers(), "Prefer": "return=minimal"},
                          params={"id": f"eq.{robot_id}"})

    # === 任务管理 ===
    def fetch_pending_tasks(self, robot_id: str):
        return httpx.get(f"{self.base_url}/api/rest/tasks",
                        params={
                            "select": "*",
                            "robot_id": f"eq.{robot_id}",
                            "status": "eq.pending",
                            "order": "priority.desc,created_at.asc"
                        },
                        headers=self._headers())

    def update_task_status(self, task_id: str, status: str):
        return httpx.patch(f"{self.base_url}/api/rest/tasks",
                          json={"status": status},
                          headers={**self._headers(), "Prefer": "return=minimal"},
                          params={"id": f"eq.{task_id}"})

    # === 执行记录 ===
    def create_execution(self, execution_data: dict):
        return httpx.post(f"{self.base_url}/api/rest/executions",
                         json=execution_data, headers=self._headers())

    def update_execution(self, execution_id: str, data: dict):
        return httpx.patch(f"{self.base_url}/api/rest/executions",
                          json=data,
                          headers={**self._headers(), "Prefer": "return=minimal"},
                          params={"id": f"eq.{execution_id}"})

    def log_execution(self, execution_id: str, level: str, message: str, component: str = ""):
        return httpx.post(f"{self.base_url}/api/rest/execution_logs",
                         json={
                             "execution_id": execution_id,
                             "level": level,
                             "message": message,
                             "component_name": component
                         },
                         headers=self._headers())

    # === 文件操作 ===
    def download_script(self, path: str, local_path: str):
        resp = httpx.get(f"{self.base_url}/api/storage/objects/rpa-resources/{path}",
                        headers=self._headers())
        with open(local_path, "wb") as f:
            f.write(resp.content)
        return local_path

    def upload_result(self, path: str, file_path: str):
        with open(file_path, "rb") as f:
            return httpx.post(
                f"{self.base_url}/api/storage/objects/rpa-resources/{path}",
                files={"file": f},
                headers={"Authorization": f"Bearer {self.access_token}"}
            )


# 全局单例
insforge = InsForgeClient()
```

## 5. 数据迁移：MySQL → PostgreSQL

### 5.1 方案选择

| 方案 | 工具 | 适用场景 | 推荐度 |
|------|------|----------|--------|
| **pgloader** | 开源 ETL 工具 | 全量迁移 + 类型自动转换 | 推荐 |
| 手动导出导入 | mysqldump + sed + psql | 小数据量、需精细控制 | 备选 |
| 编写迁移脚本 | Python/Node.js | 需要数据清洗转换 | 补充 |

### 5.2 pgloader 迁移步骤

```bash
# 1. 安装 pgloader
apt-get install pgloader

# 2. 创建迁移配置文件 migrate.load
cat > migrate.load << 'EOF'
LOAD DATABASE
  FROM mysql://root:password@172.16.100.211:3306/astron_rpa
  INTO postgresql://postgres:xqxatcdj@172.16.100.211:5432/insforge

WITH
  create_tables = true,
  create_indexes = true,
  reset_sequences = true,
  disable_triggers = true

ALTER SCHEMA 'astron_rpa' RENAME TO 'public'

CAST
  type datetime to timestamptz drop default drop not null using zero-dates-to-null,
  type tinyint to boolean using tinyint-to-boolean,
  type json to jsonb;
EOF

# 3. 执行迁移
pgloader migrate.load

# 4. 验证数据完整性
psql -h 172.16.100.211 -U postgres -d insforge -c "
SELECT
  'robots' AS table_name, COUNT(*) AS row_count FROM public.robots
UNION ALL SELECT 'projects', COUNT(*) FROM public.projects
UNION ALL SELECT 'workflows', COUNT(*) FROM public.workflows
UNION ALL SELECT 'tasks', COUNT(*) FROM public.tasks
UNION ALL SELECT 'executions', COUNT(*) FROM public.executions;
"
```

### 5.3 注意事项

1. **MySQL 的 `TINYINT(1)` → PostgreSQL `BOOLEAN`**：pgloader 的 `tinyint-to-boolean` 会自动处理
2. **`AUTO_INCREMENT` → `SERIAL`/`IDENTITY`**：pgloader 自动转换
3. **`DATETIME` → `TIMESTAMPTZ`**：注意时区转换
4. **`ENUM` 类型**：PostgreSQL 不支持，需手动改为 `VARCHAR` + CHECK 约束
5. **JSON 字段**：MySQL `JSON` → PostgreSQL `JSONB`（性能更好）

## 9. 风险与回滚策略

### 9.1 风险矩阵

| 风险 | 影响 | 概率 | 缓解措施 |
|:---|:---|:---|:---|
| **数据迁移丢失** | 高 | 中 | 迁移前全量备份 MySQL；pgloader 验证；逐表校验行数 |
| **API 兼容性** | 中 | 中 | 保留旧 API 接口的适配层；灰度切换 |
| **引擎通信中断** | 高 | 低 | 引擎侧增加重试机制；本地缓存任务队列 |
| **WebSocket 迁移** | 中 | 中 | 先做双通道（同时连接 WS 和 Socket.IO），验证后切换 |
| **性能退化** | 低 | 低 | PostgREST 性能不输 Spring Boot；压测验证 |
| **Anon Key 泄露** | 高 | 低 | 通过环境变量注入，不提交到代码仓库 |

### 9.2 回滚方案

**每阶段可独立回滚：**

| 阶段 | 回滚方式 |
|:---|:---|
| 阶段一（基础设施） | 无破坏性操作，无需回滚 |
| 阶段二（API 替换） | 保留 `src/api/http/` 目录，通过环境变量 `VITE_USE_INSFORGE=false` 切换 |
| 阶段三（引擎适配） | 引擎侧通过配置开关选择旧 API 或新 API |
| 阶段四（下线旧服务） | Docker 容器先 `stop` 不 `rm`，验证通过后再清理 |

### 10.2 执行日志实时推送完整示例

```typescript
// frontend/packages/web-app/src/views/actuator/useExecutionRealtime.ts
import { ref, onUnmounted } from 'vue';
import { realtime, db } from '@/insforge';

export function useExecutionRealtime(executionId: string) {
  const logs = ref<any[]>([]);
  const status = ref<string>('running');
  let channel: any = null;

  async function subscribe() {
    // 1. 加载已有日志
    const { data } = await db.executionLogs()
      .select('*')
      .eq('execution_id', executionId)
      .order('logged_at', { ascending: true });
    logs.value = data || [];

    // 2. 订阅实时更新
    channel = realtime.executionChannel();

    // 监听新日志
    channel.on('postgres_changes', {
      event: 'INSERT',
      schema: 'public',
      table: 'execution_logs',
      filter: `execution_id=eq.${executionId}`
    }, (payload: any) => {
      logs.value.push(payload.new);
    });

    // 监听执行状态变更
    channel.on('postgres_changes', {
      event: 'UPDATE',
      schema: 'public',
      table: 'executions',
      filter: `id=eq.${executionId}`
    }, (payload: any) => {
      status.value = payload.new.status;
      if (payload.new.status === 'completed' || payload.new.status === 'failed') {
        unsubscribe();
      }
    });

    channel.subscribe();
  }

  function unsubscribe() {
    if (channel) {
      channel.unsubscribe();
    }
  }

  onUnmounted(unsubscribe);

  return { logs, status, subscribe };
}
```
