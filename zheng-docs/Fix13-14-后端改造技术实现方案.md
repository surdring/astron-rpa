## 一、Fix-13: 引擎企业共享文件迁移

### 1.1 现状分析

**当前文件**: [`engine/components/astronverse-enterprise/src/astronverse/enterprise/enterprise.py`](file:///d:/develop/astron-rpa-bak/engine/components/astronverse-enterprise/src/astronverse/enterprise/enterprise.py)

| 方法 | 当前调用 | 依赖服务 |
|------|----------|----------|
| `upload_to_sharefolder` | `POST /api/resource/file/shared-file-upload` | resource-service |
| `upload_to_sharefolder` | `POST /api/robot/robot-shared-file/addSharedFileInfo` | robot-service |
| `download_from_sharefolder` | `GET /api/resource/file/download` | resource-service |
| `get_remote_var_key` | `GET /api/robot/robot-shared-var/shared-var-key` | robot-service |
| `get_remote_var_value` | `POST /api/robot/robot-shared-var/get-batch-shared-var` | robot-service |

当前流程：
1. 文件先上传到 resource-service，返回 `fileid`
2. 再将文件元信息（fileId/fileType/fileName）提交到 robot-service 的 `addSharedFileInfo`
3. 下载时通过 `fileId` 向 resource-service 请求文件流

---

### 1.2 改造目标

- **文件存储**：迁移到 InsForge Storage（`rpa-shared` bucket）
- **文件元数据**：迁移到 PostgreSQL 的 `shared_files` 表
- **共享变量**：迁移到 PostgreSQL 的 `shared_variables` 表
- **移除依赖**：不再调用 resource-service 和 robot-service
- **统一 SDK**：使用 `engine/shared/insforge_client.py` 中的 `insforge` 单例

#### 1.4.2 改造 `upload_to_sharefolder`

```python
@staticmethod
@atomicMg.atomic(
    "Enterprise",
    inputList=[
        atomicMg.param(
            "file_path",
            formType=AtomicFormTypeMeta(
                type=AtomicFormType.INPUT_VARIABLE_PYTHON_FILE.value,
                params={"filters": [], "file_type": "file"},
            ),
        ),
    ],
    outputList=[atomicMg.param("upload_result", types="Str")],
)
def upload_to_sharefolder(file_path: PATH = ""):
    """Upload file to InsForge Storage and record metadata in shared_files table."""
    logger.info("upload_to_sharefolder start: path=%s", file_path)
    if not (os.path.exists(file_path) and os.path.isfile(file_path)):
        return BaseException(PATH_INVALID_FORMAT.format(file_path), "请重新输入正确的文件路径")

    try:
        file_name = os.path.basename(file_path)
        storage_path = "shared/{}/{}".format(_insforge_user_id() or "common", file_name)

        # 1. 上传到 InsForge Storage
        logger.info("upload_to_sharefolder uploading: bucket=%s path=%s", SHARED_FILES_BUCKET, storage_path)
        upload_resp = insforge.upload_file(SHARED_FILES_BUCKET, storage_path, file_path)
        logger.info("upload_to_sharefolder upload response: status=%s", upload_resp.status_code)
        if upload_resp.status_code not in (200, 201):
            raise BaseException(
                FILE_UPLOAD_FAILED_FORMAT.format(upload_resp.text),
                "文件上传失败，请检查 Storage 服务！",
            )

        # 2. 写入 shared_files 表
        file_size = os.path.getsize(file_path)
        file_ext = os.path.splitext(file_name)[1].lstrip(".").lower()
        record = {
            "file_name": file_name,
            "file_path": storage_path,
            "file_size": file_size,
            "file_type": file_ext,
            "storage_bucket": SHARED_FILES_BUCKET,
            "tags": [],
        }
        logger.info("upload_to_sharefolder insert record: %s", record)
        ins_resp = insforge.insert_record("shared_files", record)
        logger.info("upload_to_sharefolder insert response: status=%s", ins_resp.status_code)
        if ins_resp.status_code not in (200, 201):
            raise BaseException(
                FILE_UPLOAD_FAILED_FORMAT.format(ins_resp.text),
                "文件已上传但元数据写入失败！",
            )

        return "上传成功"
    except Exception as e:
        logger.exception("upload_to_sharefolder error: %s", e)
        raise BaseException(FILE_UPLOAD_FAILED_FORMAT.format(e), "")
```

#### 1.4.3 改造 `download_from_sharefolder`

原方法参数 `file_path` 实际含义是 `fileId`（int 类型）。改造后改为通过 `file_path`（存储路径）或 `file_id`（UUID）下载。

```python
@staticmethod
@atomicMg.atomic(
    "Enterprise",
    inputList=[
        atomicMg.param(
            "file_path",
            formType=AtomicFormTypeMeta(type=AtomicFormType.REMOTEFOLDERS.value),
        ),
        atomicMg.param(
            "save_folder",
            formType=AtomicFormTypeMeta(
                type=AtomicFormType.INPUT_VARIABLE_PYTHON_FILE.value,
                params={"filters": [], "file_type": "folder"},
            ),
        ),
    ],
    outputList=[atomicMg.param("download_result", types="Str")],
)
def download_from_sharefolder(file_path: str, save_folder: PATH = ""):
    """Download file from InsForge Storage by storage path or shared_files id."""
    logger.info("download_from_sharefolder start: file_path=%s save_folder=%s", file_path, save_folder)
    if not Path(save_folder).is_absolute():
        raise Exception("文件夹路径错误：{} 不是绝对路径".format(save_folder))
    if not os.path.exists(save_folder):
        os.makedirs(save_folder)
    if not os.path.isdir(save_folder):
        raise Exception("文件夹路径错误：{} 不是文件夹路径".format(save_folder))

    try:
        # file_path 可能是 shared_files.id（UUID）或 storage_path
        storage_path = file_path
        file_name = None
        if len(file_path) == 36 and file_path.count("-") == 4:
            resp = insforge.query_table(
                "shared_files",
                select="file_name,file_path",
                filters={"id": file_path},
                single=True,
            )
            logger.info("download_from_sharefolder query record: status=%s", resp.status_code)
            if resp.status_code == 200:
                meta = resp.json()
                storage_path = meta.get("file_path")
                file_name = meta.get("file_name")

        if not storage_path:
            raise Exception("未找到对应的共享文件")

        if not file_name:
            file_name = os.path.basename(storage_path)

        save_path = os.path.join(save_folder, file_name)
        if os.path.exists(save_path):
            base, ext = os.path.splitext(file_name)
            count = 1
            while os.path.exists(save_path):
                new_filename = "{}({}){}".format(base, count, ext)
                save_path = os.path.join(save_folder, new_filename)
                count += 1

        logger.info("download_from_sharefolder downloading: bucket=%s path=%s -> %s", SHARED_FILES_BUCKET, storage_path, save_path)
        download_resp = insforge.download_file(SHARED_FILES_BUCKET, storage_path, save_path)
        logger.info("download_from_sharefolder download response: status=%s", download_resp.status_code)
        if download_resp.status_code != 200:
            raise BaseException(
                FILE_DOWNLOAD_FAILED_FORMAT.format(download_resp.text),
                "请检查下载接口！",
            )

        logger.info("download_from_sharefolder success: %s", save_path)
        return save_path
    except Exception as e:
        logger.exception("download_from_sharefolder error: %s", e)
        raise BaseException(FILE_UPLOAD_FAILED_FORMAT.format(e), "")
```

```python
def get_remote_var_key() -> str:
    """获取当前用户的共享变量主密钥。"""
    resp = insforge.query_table(
        "shared_variables",
        select="var_key",
        filters={"var_name": "__master_key__"},

    logger.info("get_remote_var_key response: status=%s", resp.status_code)
    if resp.status_code == 200:
        data = resp.json()
        return data.get("var_key", "")
    return ""


def get_remote_var_value(key: str) -> dict:
    """根据 key 获取共享变量值。"""
    resp = insforge.query_table(
        "shared_variables",
        select="sub_vars,encrypt",
        filters={"var_key": key},
        single=True,
    )
    logger.info("get_remote_var_value response: key=%s status=%s", key, resp.status_code)
    if resp.status_code == 200:
        data = resp.json()
        return {
            "subVarList": data.get("sub_vars", []),
            "encrypt": data.get("encrypt", False),
        }
    return None
```

### 2.2 浏览器插件 RPC 改造

#### 2.2.1 推荐方案：直连 browser-bridge

**原因**：
- browser-bridge 已是独立服务，原生提供 `/browser/transition`
- 保留了 WebSocket 管理、超时控制、错误处理等能力
- 改动最小，无需修改浏览器插件

**改造实现**：

```python
import os
from urllib.parse import urljoin


def _browser_bridge_url():
    """browser-bridge HTTP 服务地址。"""
    return os.environ.get("BROWSER_BRIDGE_URL") or "http://127.0.0.1:19082"


class Browser:
    # ... 其他方法保持不变 ...

    @staticmethod
    def send_browser_rpc(req: dict, timeout: float = 0.0) -> Any:
        """发送浏览器RPC请求，直连 browser-bridge。"""
        url = _browser_bridge_url()
        logger.info("send_browser_rpc request: url=%s/broswer/transition timeout=%s req=%s", url, timeout, json.dumps(req))
        try:
            res = requests.post(
                "{}/browser/transition".format(url),
                json=req,
                timeout=timeout or 10,
            )
            logger.info("send_browser_rpc response: status=%s body=%s", res.status_code, res.text[:200])
            return res
        except Exception as e:
            logger.exception("send_browser_rpc error: %s", e)
            raise
```

#### 2.2.2 备选方案：直连浏览器插件

### 2.3 trigger WebSocket 改造

#### 2.3.1 推荐方案：InsForge Realtime（Socket.IO）

**设计思路**：
- 每个 trigger server 启动时，用当前用户的 user_id 订阅一个 realtime channel（如 `remote_control:{user_id}`）
- 前端/云端通过 InsForge Realtime 向该 channel 发送消息
- trigger 收到消息后执行 `remote_run` / `remote_stop_current`
- 执行结果也通过 realtime 返回给发送方

**为什么不用 openapi-service**：
- openapi-service 当前未启动，且依赖 MySQL
- 轻量化方向是去 Java/Python 服务，统一用 InsForge BaaS

#### 2.3.2 依赖安装

trigger server 需要 Socket.IO 客户端。检查 `pyproject.toml` 是否已有 `python-socketio[client]`：

```toml
dependencies = [
    "python-socketio[client]>=5.0",
]
```

如果只有 `python-socketio[asyncio_client]` 也可以。

#### 2.3.3 新增 InsForge Realtime 客户端

在 trigger server 中新增 `realtime_client.py`：

```python
import os
import socketio
from astronverse.trigger.core.logger import logger
from astronverse.trigger.server.gateway_client import execute_single_project, send_stop_current, get_executor_status

INSFORGE_URL = os.environ.get("INSFORGE_API_URL") or "http://172.16.100.211:7130"
JWT_TOKEN = os.environ.get("INSFORGE_JWT_TOKEN") or ""


class TriggerRealtimeClient:
    """通过 InsForge Realtime 接收远程控制命令。"""

    def __init__(self, user_id: str):
        self.user_id = user_id
        self.channel = f"remote_control:{user_id}"
        self.sio = socketio.Client(logger=logger, engineio_logger=False)
        self._register_events()

    def _register_events(self):
        @self.sio.event
        def connect():
            logger.info("trigger realtime connected, join channel=%s", self.channel)
            self.sio.emit("subscribe", {"channel": self.channel})

        @self.sio.event
        def disconnect():
            logger.info("trigger realtime disconnected")

        @self.sio.on("remote.run")
        def on_remote_run(data):
            logger.info("remote.run received: %s", data)
            for i in range(3):
                if not get_executor_status():
                    result = execute_single_project(data)
                    self.sio.emit("remote.run.reply", {"channel": self.channel, "data": result})
                    return
                import time
                time.sleep(2)
            self.sio.emit("remote.run.reply", {"channel": self.channel, "data": {"code": "5001", "msg": "有任务在运行中", "data": None}})

        @self.sio.on("remote.stop_current")
        def on_remote_stop(data):
            logger.info("remote.stop_current received: %s", data)
            if get_executor_status():
                result = send_stop_current()
            else:
                result = {"code": "5001", "msg": "没有任务在运行中", "data": None}
            self.sio.emit("remote.stop_current.reply", {"channel": self.channel, "data": result})

    def start(self):
        try:
            url = INSFORGE_URL.replace("http://", "ws://").replace("https://", "wss://")
            logger.info("trigger realtime connecting: %s", url)
            self.sio.connect(
                url,
                headers={"Authorization": f"Bearer {JWT_TOKEN}"},
                namespaces="/realtime",
                wait_timeout=10,
            )
            logger.info("trigger realtime started")
        except Exception as e:
            logger.exception("trigger realtime start failed: %s", e)

    def stop(self):
        try:
            if self.sio.connected:
                self.sio.disconnect()
            logger.info("trigger realtime stopped")
        except Exception as e:
            logger.exception("trigger realtime stop failed: %s", e)
```

#### 2.3.4 改造 `WebSocketManager`

在 `server.py` 中：

```python
from astronverse.trigger.server.realtime_client import TriggerRealtimeClient


class WebSocketManager:
    def __init__(self):
        self.realtime_client: Optional[TriggerRealtimeClient] = None
        self._thread = None
        self._stop_event = threading.Event()

    def start(self):
        try:
            if self.realtime_client and self.realtime_client.sio.connected:
                logger.info("Realtime 管理器已在运行")
                return
            self._stop_event.clear()

            # 从 app_context 或环境变量获取 user_id
            user_id = os.environ.get("INSFORGE_USER_ID") or getattr(app_context, "user_id", None)
            if not user_id:
                logger.error("未配置 INSFORGE_USER_ID，无法启动远程控制 realtime")
                return

            self.realtime_client = TriggerRealtimeClient(user_id)
            self.realtime_client.start()
            logger.info("Realtime 管理器启动成功")
        except Exception as e:
            logger.error(f"Realtime 管理器启动失败: {e}")

    def stop(self):
        try:
            if self.realtime_client:
                self.realtime_client.stop()
            self.realtime_client = None
            logger.info("Realtime 管理器已停止")
        except Exception as e:
            logger.error(f"Realtime 管理器停止失败: {e}")
```

#### 2.3.5 前端/云端发送远程控制命令

前端通过 InsForge Realtime SDK 发送命令：

```typescript
import { insforge } from '@rpa/shared'

const channel = `remote_control:${user_id}`
const realtime = insforge.realtime

realtime.on('remote.run.reply', (payload) => {
  console.log('remote run result:', payload)
})

await realtime.subscribe(channel)
await realtime.send({
  channel,
  event: 'remote.run',
  payload: { robotId: 'xxx', params: {} },
})
```

或者通过 Edge Function 转发（云端场景）：

```typescript
// functions/remote-control/index.ts
Deno.serve(async (req) => {
  const { user_id, action, data } = await req.json()
  const client = createClient({ ... })
  await client.realtime.send({
    channel: `remote_control:${user_id}`,
    event: action, // remote.run | remote.stop_current
    payload: data,
  })
  return Response.json({ success: true })
})
```

## 四、风险与注意事项

### 4.1 Fix-13 风险

1. **`file_path` 参数语义变化**
   - 原 `download_from_sharefolder` 接收 int 型 `fileId`
   - 改造后接收 UUID（`shared_files.id`）或 storage path
   - 需要在调用方统一调整，或增加兼容转换

2. **文件名冲突**
   - InsForge Storage 的 `upload_file` 如果路径已存在可能覆盖
   - 建议上传路径中包含 UUID 前缀：`shared/{user_id}/{uuid}-{file_name}`

3. **Storage bucket 权限**
   - `rpa-shared` 应设置为 private，下载时需要认证
   - `insforge_client.py` 的 `download_file` 已经带 JWT token

### 4.2 Fix-14 风险

1. **browser-bridge 端口变更**
   - 原网关统一走 `13159`，现在直接访问 `19082`
   - 如果用户自定义了 browser-bridge 端口，需同步配置 `BROWSER_BRIDGE_URL`

2. **Realtime 连接稳定性**
   - Socket.IO 客户端需要处理断线重连
   - 当前 `TriggerRealtimeClient` 示例没有自动重连，可配置 `reconnection=True`

3. **JWT Token 注入**
   - trigger server 作为后台服务，需要持久化的 JWT token
   - 考虑使用 refresh token 机制或从本地安全存储读取

4. **兼容旧 openapi-service**
   - 如果仍有其他组件调用 openapi-service 的 `/ws`，需要同步迁移
   - 建议完全废弃 openapi-service 的 WebSocket 能力
