### Issue-6: Vite dev server 缓存导致 shared 包变更不生效

**优先级**: P3  
**状态**: ⏳ 持续关注

**现象**:
- 修改 `shared/src/api-client.ts` 后，重建 `shared` 包（`pnpm build`），但 Vite dev server 仍提供旧版本
- 需要重启 Vite dev server 才能生效

**原因**:
- Vite 对 workspace 依赖的模块有缓存机制
- `shared` 包的 `postinstall` 钩子只在 `pnpm install` 时触发

**建议**:
- 修改 `shared` 包后，执行 `pnpm build` 然后重启 dev server
- 或使用 `pnpm dev --force` 清除 Vite 缓存

## 客户端启动顺序（完整流程）

```bash
# 1. 构建 tokens（生成 tailwind-preset.js 等）
cd frontend/packages/shared && pnpm build:tokens

# 2. 构建 SDK（生成 client-sdk.js）
cd frontend/packages/electron-app && pnpm build:sdk

# 3. 构建 shared 包（更新 dist 文件）
cd frontend/packages/shared && pnpm build

# 4. 启动 Vite dev server
cd frontend/packages/web-app && pnpm dev
```
